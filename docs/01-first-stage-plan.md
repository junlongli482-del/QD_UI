# 第一阶段方案：原版还原 Demo 与移动端适配梳理

## 背景

现有项目是教学类在线平台，当前需要进行重新设计。已确认的核心问题包括手机端 UI 显示不佳、文字或图标乱码、页面适配不足，同时网页端 UI 也需要重新设计。

本阶段先不直接做完整 UI 改版，而是先基于现有平台做原版还原 Demo，确保功能模块、页面结构和关键流程没有遗漏。

## 目标

1. 还原现有平台主要页面和功能入口，形成可在浏览器查看的纯前端 Demo。
2. 梳理 PC 端和手机端的页面结构、导航结构、功能模块和关键流程。
3. 记录手机端兼容性问题，包括布局、文字、图标、导航、浮动入口等。
4. 为后续 UI 改版、UI 图设计和开发交付文档提供基础。

## 本次范围

第一版 Demo 覆盖以下范围：

- 登录页
- 工作台首页
- 作业管理
- 智能组卷
- 数字校本
- 学情分析
- 数据大屏
- 备课中心
- 阅卷考试
- 学校管理
- 个人中心入口
- 右侧辅助入口：试题篮、问题反馈、使用说明、移动端

## 暂不包含

- 不接真实后端接口。
- 不做真实数据提交。
- 不做完整权限系统。
- 不做生产级业务逻辑。
- 不先进行大幅视觉改版。
- 不保存平台账号密码到仓库。

## 完成标准

第一阶段完成时，应满足：

1. 纯前端 Demo 可以在浏览器中打开并查看。
2. PC 端和手机端都有可查看的布局效果。
3. 主导航和首页功能模块完整，不遗漏已识别的一级模块和主要入口。
4. 手机端问题已形成清单，并能对应到页面或组件。
5. 后续开发人员可以根据文档理解：哪些页面要还原、哪些问题要修复、哪些内容暂不实现。
