# 教学平台 UI 重设计与移动端适配需求说明书

## 1. 项目背景

现有项目是一个教学类在线平台，当前需要进行重新设计。项目方不直接进行真实开发，也没有现有源码，主要通过产品文档、UI 图和前端 Demo 配合对方开发人员落地。

本阶段需要先基于现有平台做一个纯前端 Demo 或原型展示，用于给领导查看，并作为后续 UI 改版和开发交付的基础。

当前已访问平台：

- 平台名称：金版学案智慧教学平台
- 平台地址：https://eduaii.com/
- 当前登录身份：李老师测试
- 当前学段学科：高中 / 语文
- 当前学年：26-27

## 2. 项目目标

本项目目标分为两个层次：

### 2.1 第一阶段目标

先做一个“原版还原 Demo”，尽量还原现有平台页面结构、功能入口、模块关系和基础视觉样式，确保原有功能模块不丢失。

第一阶段重点：

- 还原平台主要页面和模块入口。
- 梳理 PC 端和手机端页面结构。
- 记录并解决手机端基础显示问题。
- 形成可给领导查看的纯前端 Demo。
- 为后续 UI 改版和开发交付文档提供基础。

### 2.2 后续目标

在原版还原 Demo 确认后，再进行新版 UI 设计和移动端适配优化。

后续重点：

- 完成 PC 端 UI 重新设计。
- 完成手机端适配设计。
- 输出 UI 图、页面说明、组件规范和开发验收标准。
- 配合对方开发人员进行实现。

## 7. Demo 要求

### 7.1 Demo 定位

Demo 是纯前端展示版本，用于领导评审和开发沟通，不作为生产系统。

### 7.2 Demo 形式

Demo 应支持：

- 手机端布局查看。
- 模拟登录。
- 模块入口点击。
- 二级页面占位展示。

### 7.3 页面还原要求

必须还原：

- 页面名称
- 一级模块
- 二级入口名称
- 顶部学年和用户信息
- 试题篮数量展示

可以使用模拟数据：

### 7.4 二级页面要求

- 可返回工作台。
- 不要求真实业务数据。

## 8. 移动端适配要求

移动端是本项目重点。

### 8.1 已观察视口

- 手机视口：375 x 812

### 8.2 已发现问题

#### 图标字体显示异常

现象：

- 登录页和登录后页面中，多个图标显示为异常字符。

要求：

- Demo 中不能出现图标乱码。
- 后续开发需检查图标字体资源路径、跨域、缓存和 MIME 类型。

#### 左侧导航过窄

现象：

- 手机端左侧导航被压缩到约 47px。
- 导航可识别性不足。

要求：

- 手机端不能直接照搬 PC 端窄侧栏。
- 移动端导航应改为抽屉、顶部菜单、底部导航或其他适合触屏的形式。
- 功能入口不能丢失。

#### 顶部用户区域空间紧张

现象：

- 手机端顶部仍展示学年和用户名，空间较紧张。

要求：

- 长用户名不能溢出。
- 用户入口要清晰。
- 可将用户信息收纳到头像或个人中心菜单。

#### 右侧浮动工具条挤占内容

现象：

- 右侧试题篮、问题反馈、使用说明、移动端入口贴边显示，容易挤占内容。

要求：

- 手机端浮动入口不能遮挡主内容。
- 可改为折叠悬浮按钮或更多菜单。

#### 首页卡片偏 PC 结构

现象：

- 手机端虽然可单列展示，但整体仍是 PC 卡片思路。

要求：

- 手机端模块入口要适合触控。
- 入口文字不重叠、不截断核心含义。
- 页面不出现横向滚动。

## 9. PC 端 UI 要求

第一阶段 PC 端以还原为主。

要求：

- 保留顶部栏、左侧导航、内容区的基本结构。
- 保留首页模块卡片结构。
- 保留一级模块和二级入口名称。
- 不先进行大幅视觉改版。

后续 UI 改版阶段再进一步设计：

- 新版配色
- 字体规范
- 按钮规范
- 表格规范
- 弹窗规范
- 空状态和异常状态
- PC 与手机端统一视觉体系

## 10. 验收标准

第一阶段完成标准：

1. 纯前端 Demo 可以在浏览器中打开并查看。
2. 登录页可以模拟进入工作台。
3. PC 端工作台首页结构完整。
4. 手机端可以正常浏览核心页面。
5. 一级模块和主要二级入口不遗漏。
6. 点击模块入口后有对应页面或占位页。
7. 页面不接真实接口、不提交真实数据。
8. 手机端不出现横向滚动。
9. 手机端文字不重叠、不遮挡、不丢失核心含义。
10. 图标不显示为乱码字符。
11. 右侧辅助入口不遮挡主内容。
12. 文档能让 UI 设计师、前端 Demo 人员和对方开发人员理解本阶段要做什么。
