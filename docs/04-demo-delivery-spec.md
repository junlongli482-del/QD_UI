# 原型 Demo 交付说明

## Demo 定位

Demo 是用于领导评审和开发沟通的纯前端展示版本，不是生产系统。

第一版 Demo 的核心价值是：

- 还原原平台模块。
- 证明功能入口没有遗漏。
- 暴露并修复移动端显示问题。
- 为下一步 UI 改版提供可对比基础。

## 页面要求

### 登录页

需要展示：

- 平台名称
- 密码登录
- 用户名输入
- 密码输入
- 登录按钮

交互要求：

- 可点击登录进入工作台。
- 使用模拟登录，不连接真实接口。

### 工作台首页

需要展示：

- 顶部学年和用户区域
- 左侧或移动端导航
- 首页模块卡片
- 右侧辅助入口或移动端收纳入口

模块必须包含：

- 作业管理
- 智能组卷
- 数字校本
- 学情分析
- 数据大屏
- 备课中心
- 阅卷考试
- 学校管理

### 二级页面

第一版可使用结构化占位页。

要求：

- 点击模块入口后能进入对应页面。
- 页面标题、所属模块和入口名称正确。
- 可返回工作台。
- 不要求真实业务数据。

## 响应式要求

PC 端：

- 保留原平台左侧导航 + 顶部栏 + 内容区的基本结构。
- 首页模块以卡片方式展示。

手机端：

- 不照搬 PC 端窄侧栏。
- 导航需要改为适合触屏的结构。
- 模块入口需要完整保留。
- 浮动工具入口不能遮挡主内容。

## 内容还原要求

必须还原：

- 页面名称
- 一级模块
- 二级入口名称
- 顶部学年和用户信息
- 试题篮数量展示

可以使用模拟数据：

- 列表内容
- 统计数字
- 学情图表
- 试卷、作业、资源名称

## 不做内容

- 不连接真实接口。
- 不提交作业、试卷、考试等真实数据。
- 不实现真实组卷逻辑。
- 不实现真实阅卷流程。
- 不保存真实账号密码。

## 给开发团队的后续交付物

第一阶段之后建议补齐：

- 页面清单
- 组件清单
- 移动端适配规则
- 断点规则
- 字体和图标规则
- 颜色和按钮规范
- 表格、筛选、弹窗、空状态规范
- 验收标准

## 备注

当前文档只记录已确认和已观察内容。未进入的二级页面，需要后续继续访问确认后再补充。
