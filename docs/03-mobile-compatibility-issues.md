# 移动端兼容性问题清单

## 观察视口

当前已按手机视口 375 x 812 观察工作台首页。

## 已发现问题

### 1. 图标字体显示异常

现象：

- 登录页和登录后页面中，多个图标显示为类似乱码的私有字符。
- 示例表现包括：登录页出现异常图标字符，导航和顶部区域也存在类似字符。

影响：

- 用户无法识别图标含义。
- 页面显得不完整，影响专业感。

建议：

- 明确图标字体加载来源。
- 如果是字体文件加载失败，需要检查字体资源路径、跨域、缓存和 MIME 类型。
- Demo 阶段可先使用标准图标库或占位图标表达原有含义。

### 2. 手机端左侧导航过窄

现象：

- 手机视口下左侧导航被压缩到约 47px。
- 学科和导航入口以极窄区域展示，信息表达不足。

影响：

- 导航识别困难。
- 触控区域偏小。
- 不利于移动端使用。

建议：

- 手机端改为顶部菜单、抽屉导航或底部导航。
- 保留一级模块完整性，避免功能入口丢失。

### 3. 顶部用户区域空间紧张

现象：

- 手机视口下顶部仍展示学年和用户名。
- 右侧区域宽度有限，用户信息靠右紧贴。

影响：

- 长用户名或更多状态信息容易溢出。
- 用户操作入口不够清晰。

建议：

- 手机端顶部只保留核心状态。
- 用户信息可放入头像菜单或个人中心入口。

### 4. 右侧浮动工具条挤占内容

现象：

- 右侧辅助入口在手机端贴近屏幕右侧。
- 试题篮、问题反馈、使用说明、移动端等入口占据内容区域边缘。

影响：

- 容易遮挡内容。
- 触控体验不稳定。
- 小屏上视觉负担较重。

建议：

- 手机端改为折叠悬浮按钮。
- 非核心入口可收纳到更多菜单。

### 5. 首页卡片虽然可缩放，但仍偏 PC 思路

现象：

- 首页模块卡片在手机端缩为单列展示。
- 卡片和标签尺寸变小，但信息架构仍沿用 PC 端。

影响：

- 可以浏览，但不一定适合高频移动操作。
- 模块入口较多时，用户需要长距离滚动。

建议：

- 手机端可使用模块列表、宫格入口或分组折叠。
- 优先展示常用模块，低频模块放入更多入口。

## 第一阶段验收标准

第一版 Demo 中，手机端至少应做到：

- 页面不出现横向滚动。
- 文字不重叠、不截断核心含义。
- 图标不显示为乱码字符。
- 导航入口可识别、可点击。
- 右侧浮动入口不遮挡主内容。
- PC 端和手机端功能模块数量一致。
